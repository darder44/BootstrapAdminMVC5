#其中 type 的值可以有
#    feat: 新功能 
#    fix: 修復bug
#    doc: 文檔改變
#    style: 代碼格式改變
#    refactor: 某個已有功能重構
#    perf: 性能優化
#    test: 增加測試
#    build: 改變了build工具 如 更新 props 文件
#    revert: 撤銷上一次的 commit 
#    db: 資料庫文件或者腳本更改
#    script: 輔助腳本改變
#
#scope: 用来說明此次修改的影响范围
#     all: 表示影响面大 ，如修改了网络框架  會对真個程式产生影响
#     location: 表示影响小，某個小小的功能
#     module: 表示會影响某個模塊 如登入模塊、首頁模塊 、用户管理模塊等等
#
#subject: 用来簡要描述本次改动，概述就好了
# 
#body: 具体的修改信息 應該尽量詳詳
# 
#footer: 放置写备注啥的，如果是 bug ，可以把bug id放入
